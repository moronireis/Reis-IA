Linear Design System

Visual design tokens and patterns extracted from linear.app. Reference for Reis IA dark-mode premium design. Not a template -- a study in restraint, opacity, and typographic hierarchy.

Signature Elements

The 3-5 things that make Linear feel distinctly Linear. Each includes an interactive demo and adaptation notes for Reis IA.

1. Binary Grid Animation

Linear's most distinctive visual element. A 5x5 matrix of dots that animate with discrete opacity steps, creating a scanning/pulsing pattern that feels computed, not designed. The steps(1, end) timing function creates instant transitions -- dots snap between dim (0.3) and bright (1.0) with no interpolation. Two independent cycle lengths (3200ms and 2800ms) ensure the combined pattern never exactly repeats.

Reis IA Adaptation

Apply the binary-state animation concept to brand motifs. For the hourglass: sand particles that snap between positions in an hourglass-shaped grid. For strategy sections: a chess grid where squares pulse in calculated patterns. Keep steps(1, end) -- it reads as "machine intelligence" rather than organic motion. Use gold (#C9A84C) particles instead of purple.

2. Monochromatic Depth Through Opacity

Visual hierarchy achieved entirely through opacity of a single color (white). Five+ levels create depth without introducing new hues. The interface feels unified and calm because every text element is the same color at different transparencies.

Primary heading at 100%
Body text sitting at 70% opacity
Supporting caption at 50% opacity
Dimmed meta info at 48%
Quaternary timestamp at 35%
Reis IA Adaptation

Directly applicable. Use white at varying opacities for the entire text hierarchy on the dark background. Reserve the gold accent color (#C9A84C) exclusively for interactive elements and brand moments. Gold becomes more powerful through scarcity -- if everything is white-on-black, the rare gold element demands attention.

3. "Structure Felt, Not Seen"

Linear's core design philosophy from their refresh. Borders are at 5-8% opacity -- barely visible. No heavy divider lines between sections. Spacer components (empty space) replace visual separators. Navigation elements are dimmed below content contrast. The principle: don't compete for attention you haven't earned.

5% border
Barely there -- you sense the structure without seeing a line
8% border (default)
Present but quiet -- the standard divider weight
12% border (emphasis)
Slightly more visible -- used sparingly for emphasis
No border -- spacer only
Many sections separated by empty space alone (80-128px)
Reis IA Adaptation

This aligns perfectly with the "restraint over decoration" principle. For the consultancy site: use generous whitespace as section dividers, keep borders at 5-8% opacity, and ensure the hero headline and CTA are the loudest elements on any page. The chess grid background (3-5% opacity) should feel structural, not decorative.

4. The Warm Gray Shift

Linear deliberately shifted from cool blue-toned grays to warmer grays. The background is #08090a, not #000000. This warmth is barely perceptible but subconsciously feels more inviting than pure black.

Reis IA Adaptation

Consider #0A0A0A (current) or a slightly warmer #0B0A09 for the primary background. The gold accent (#C9A84C) harmonizes better with warm neutrals than with cool blacks. Test both side-by-side with the gold to determine which pairing feels more cohesive.

5. Variable Font as System Foundation

Inter loaded as a single Variable Font file enables continuous weight adjustment (not just 400/500/600/700 but any value like 450 or 550), reduced network payload, and the possibility of smooth weight transitions in animations.

Weight 300 -- Light
Weight 400 -- Normal (body text)
Weight 450 -- between Normal and Medium
Weight 500 -- Medium (labels, mini text)
Weight 550 -- between Medium and Semibold
Weight 600 -- Semibold (subheadings)
Weight 700 -- Bold (display only)
Reis IA Adaptation

Load Inter as a Variable Font for the Reis IA site. Use intermediate weights like 450 (subtle emphasis without full Medium) and 550 (emphasis without full Semibold) to create fine-grained hierarchy. Single file = faster loading. Preload with rel="preload" as="font" for instant rendering.
